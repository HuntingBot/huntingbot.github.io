Introduction

This page contains four independent search engines, each used for a different purpose:

  • Search for pattern lets you search for one or more patterns that match search criteria. A list of matching patterns is produced, and for each matching pattern found, many details about the pattern are displayed.
  • Search for still life lets you search for a still-life or difficult still-life in Life by its index number. For each still-life found, the synthesis can be displayed, if available.
  • Search Catagolue lets you search Catagolue by any search criteria Catagolue supports.
  • Search for rule lets you search for outer totalistic or non-totalistic rules, and converts between RLE rule strings, neighborhood descriptions, and Golly rule tables.

Search for pattern

Search for pattern is the primary focus of this page. It lets you search for patterns using a wide variety of criteria. To perform a search, select search criteria in the table at the top, press the Search button, then examine the search results below. Selecting a pattern from the search results displays details about the pattern, and shows the pattern image.

The top section contains one line, Add constraint:, with five drop-down lists containing constraints to use to qualify the search. These are:

  • CHECKS for selections determined by check boxes or multiple-selection drop-down lists.
  • COMMON for common numeric measurements.
  • UNCOMMON for less common numeric measurements.
  • BOXES for bounding box numeric measurements.
  • TEXT for textual (non-numeric) criteria, including search by pattern image or glider number.

The bottom section contains options that affect result display. Altering any of these after a search has been performed will have an immediate effect on the displayed results, without needing to perform a new search. Changes to any other constraints will not take effect until a new search is actually performed (although enabling or disabling search criteria may also affect which columns are shown in the result table).

  • Show results by: lets you choose to see the matched patterns by List, which shows a text table with one row per pattern, or by Stamp Page, which shows the patterns in a graphical table, like the patterns on other pages on this site. The Stamp view is more readable, but the List view provides much more information, and can show more results at once if large objects are involved. The default is List.
  • Show tool tips, when checked, shows tips about all controls and result fields on this page, by hovering the mouse over them for a few seconds. This is turned off by default.
  • Maximum results: When viewing results by list, this limits the number of results shown per page. This defaults to 20. (The maximum number of views displayed on the stamp view depends on the size of the largest result pattern and the Stamp zoom factor. This can be up to 10×11 for ×4 zoom, 13×15 for ×3 zoom, 20×23 for ×2 zoom, and 41×46 for ×1 zoom.)
  • Stamp zoom: When viewing results by stamp page, this sets the zoom factor to ×1, ×2, ×3, or ×4. The default is ×4
  • Sort results by: lets you choose the order to view the results. By default, this is by ascending population, then period, then name. The default criteria are also used to sort patterns within each matching group. If search results are displayed in a list, the criterion used for sorting is also displayed in the list.
  • Show number as: lets you choose what format to display numbers. (e.g. the same number shows as 1.5 under Decimal, 3/2 under Rational, and 1½ under Mixed). Integers are always just displayed as integers (e.g. 42). Irrational numbers always use decimal format. Numbers too large to display always use scientific notation, e.g. 1.25e10. Velocity is always shown as rational, if possible regardless of this setting.

The sections inbetween show selected criteria. For convenience, the most common constraints (Rule, Category, Period, and Minimum population) are pre-selected, as they are the most frequently used. Each criterion has a check box to the left; if that check box is un-selected, the criterion is removed. Note that irrelevant criteria are hidden, even if they are selected (e.g. if the Category criterion is set to only search for still-lifes, Heat, Modulus, Period, Slope, and Velocity are all irrelevant, so they are hidden and ignored).

Below the criterion table is a control bar with the following controls:

  • Search: Pressing this starts a new search using the selected criteria.
  • Reset: Pressing this resets all criteria on the page to their defaults.
  • Help: Pressing this launches this help page. This can also be done at any time by pressing the F1 key.
  • The search status and search result summary is displayed on the right.

The search normally takes one or two seconds. If you specify a particularly broad search that can find thousands of matches (e.g. Pattern name: Contains “e”, to find all patterns whose names contain the letter “e”), the search can take considerably longer, and may trigger a browser warning about a slow or unresponsive script. Please be patient.

Check

This section includes constraints that are specified by check boxes or drop-down lists.

Category specifies which category or category of patterns to search for:

  • Any finds all patterns.
  • Still-life finds still-lifes: stationary objects with a period of 1.
  • Pseudo-still-life finds pseudo-lifes: stationary pseudo-objects with a period of 1.
  • Quasi-still-life finds quasi-lifes: stationary quasi-objects with a period of 1.
  • Oscillator finds oscillators: stationary objects with periods >1.
  • Pseudo-oscillator finds pseudo-oscillators: stationary pseudo-objects with periods >1.
  • Quasi-oscillator finds quasi-oscillators: stationary quasi-objects with periods >1.
  • Spaceship finds spaceships: objects with non-zero velocities.
  • Pseudo-spaceship finds pseudo-spaceships: pseudo-objects with non-zero velocities.
  • Quasi-spaceship finds pseudo-spaceships: quasi-objects with non-zero velocities.
  • Wick-stretcher finds wick-stretchers: growing objects with a non-zero velocity and a linear growth rate, that produce a fixed number of ever-growing objects.
  • Puffer finds puffer trains: growing objects with a non-zero velocity and a linear growth rate, that produce an ever-increasing number of fixed-size objects.
  • Gun finds guns: stationary growing objects with a linear growth rate, that produce an ever-increasing number of moving objects.
  • Breeder finds breeders: objects with a super-linear growth rate.
  • Constellation finds constellations: collections of separated objects and pseudo-objects. Quasi-objects are a subset of these. Most constellations involve still-lifes and oscillators, but a few also include spaceships.
  • Methuselah finds methuselahs: small unstable objects that evolve for a long time before finally settling down.
  • Multiple… brings up a small pop-up window that allows you to select multiple categories.

If your search can include still-lifes, checking Difficult? includes only those still-lifes in the 14…18-bit range from the difficult-to-synthesize list (i.e. those that were determined by the expert system to not have trivial syntheses based on existing conversion recipes). All of these still-lifes have since been found to have syntheses, and additional recipes have since been added; however, for numbering purposes, these lists have not been changed.)

Checking Multiple colors only includes patterns that have multi-color syntheses in multi-color Life.

Checking Multiple files only shows patterns that have two or more synthesis files. This can happen in one of three circumstances:

  • Some very large (or difficult to synthesize) objects may have multiple syntheses separated into different files, or a single synthesis split into multiple parts.
  • Many methuselahs that produce large results, especially those that release multiple gliders, have one file showing multiple syntheses, and another containing only one, that allows the methuselah to be run to completion without multiple copies interfering with each other.
  • Some patterns have additional properties that are best illustrated in a separate file (e.g. the Queen bee’s ability to eat gliders in many different ways).

Checking Multiple pattern names shows only patterns that are known by two or more different listed names. This frequently happens for larger still-lifes that have descriptive names as well as their numeric designations. It also happens frequently for common objects that are independently discovered or named by different people.

For period 2 oscillators, P2 rotor selects the kind of rotors that are matched:

  • Flip-flop shows only oscillators whose rotor cells die only from under-population.
  • On-off shows only oscillators whose rotor cells die only from over-population.
  • Either shows both flip-flops and on-offs.
  • Not flip-flop, Not on-off, and Neither reverse these searches.

Phoenix selects patterns based on their survivability:

  • Phoenix shows only patterns that have no cells that survive (e.g. Life patterns that would work in a rule B3/S), such as oscillators with no stator cells. In some non-Life rules (e.g. B2/S2), there are also spaceships that are phoenixes.
  • Not Phoenix shows only patterns that are not phoenixes.
  • Any includes all patterns.

When searching for oscillators, Raging river selects the rotor size:

  • Raging river shows only oscillators whose rotors are completely contained in a 2×n rectangle.
  • Not raging river excludes raging rivers.
  • Any shows both kinds of oscillators.

When searching for oscillators, Rotor selects the rotor shape:

  • Babbling brook shows only oscillators whose rotor consists entirely of a single open-ended chain of cells.
  • Muttering moat shows only oscillators whose rotor consists entirely of a single closed-ended chain of cells.
  • Either shows both of the above.
  • Not babbling brook, Not muttering moat, and Neither exclude oscillators of the above types.
  • Any shows all kinds of oscillators.

Rule is a drop-down list that selects the rule whose patterns are searched:

  • B3/S23 (Life) is the most popular rule, and the one that has the most patterns on this site. This is the default.
  • B2/S2 and B34/S34 are explosive totalistic rules.
  • B36/S23 (HighLife) is another widely studied rule very similar to Life.
  • B36/S245 (Replicator rule) is rule with a small replicator.
  • B3/S238 (EightLife), B38/S23 (Pedestrian Life), and B38/S238 (Honey Life) are rules that are almost identical to Life.
  • Niemiec0…Niemiec5 are a group of related non-totalistic rules that include several small common spaceships, puffers, and replicators.
  • Any includes patterns from all of the above rules.

(Note that when searching for a pattern image, if a RLE pattern is entered and contains a rule specification, that rule overrides any rule specified in this section.)

Symmetry qualifies patterns by what kind of symmetry they possess:

  • C1 selects patterns that have no symmetry.
  • C2 selects patterns that have 180° rotational symmetry around a point, but have no other symmetry.
  • C4 selects patterns that have 90° rotational symmetry around a point, but have no other symmetry.
  • C2|4 selects both of the above.
  • C4 D8 selects patterns that have 90° rotational symmetry around a point, and may also be reflected in an axis.
  • C2|4 D4|8 selects patterns that have 180° rotational symmetry around a point, and may or may not also have 90° rotational symmetry and/or be reflected in an axis.
  • D2_+ selects patterns that are reflected in an orthogonal axis, but have no other symmetry.
  • D2_x selects patterns that are reflected in an diagonal axis, but have no other symmetry.
  • D2 selects both of the above.
  • D4_+ selects patterns that are reflected in two orthogonal axes, but have no other symmetry.
  • D4_x selects patterns that are reflected in two diagonal axes, but have no other symmetry.
  • D4 selects both of the above.
  • D2|4|8_+ selects patterns that are reflected in one or two orthogonal axes, and perhaps also in both diagonal axes.
  • D2|4|8_x selects patterns that are reflected in two diagonal axes, and perhaps also in both orthogonal axes.
  • D2|4|8 selects patterns that are reflected in two axes, and perhaps also in both other axes.
  • D4|8_+ selects patterns that are reflected in two orthogonal axes, and perhaps also in both diagonal axes.
  • D4|8_x selects patterns that are reflected in two diagonal axes, and perhaps also in both orthogonal axes.
  • D4|8 selects patterns that are reflected in two axes, and perhaps also in both other axes.
  • C2|4 D2|4|8 selects patterns that have at least one point or axis of symmetry.
  • Any selects all patterns, even those without symmetry.

In all of the above except C1 and D2_x, these must also be qualified by whether the point or axis of symmetry passes through the center of a cell (1), an edge between two cells (2), or the corner between four cells (4). You can select one or more of these, or Any to allow all of them.

Glide symmetry qualifies patterns by what kind of glide symmetry they possess. Glide symmetry occurs in oscillators and spaceships that appear rotated or reflected before returning to their original orientation.

  • C1 selects patterns that have no glide symmetry.
  • C2 selects patterns that have 180° rotational glide symmetry around a point, but have no other glide symmetry.
  • C4 selects patterns that have 90° rotational glide symmetry around a point, but have no other glide symmetry.
  • C2|4 selects both of the above.
  • D2_+ selects patterns that get reflected in an orthogonal axis, but have no other glide symmetry.
  • D2_x selects patterns that get reflected in an diagonal axis, but have no other glide symmetry.
  • D2 selects both of the above.
  • C2|4 D2 selects patterns with any glide symmetry.
  • Any selects all patterns, even those without glide symmetry.

In all of the above except C1 and D2_x, these must also be qualified in the same way Symmetry is qualified. The same qualifier is used for both. This is also necessary if both Symmetry and Glide Symmetry are D2_x.

Common

This section includes common numeric constraints.

Numeric constraints have a drop-down list indicating the kind of comparison to perform against an property (e.g. population, period, etc.). Most of these comparisons also require a numeric parameter on the left (y), and a few also require a numeric parameter on the right (x). The following comparisons are available:

  • Any: no constraint is applied. This is the default.
  • = y: the property must exactly equal y.
  • ≠ y: the property must be anything except y.
  • < y: the property must be less than but not equal to than y.
  • ≤ y: the property must be less than or equal to y.
  • > y: the property must be more than but not equal to y.
  • ≥ y: the property must be more than or equal to y.
  • x – y: the property lie within the range bracketed by numbers x and y inclusive (i.e. x ≤ property ≤ y or x ≥ property ≥ y).
  • x ~ y: the property must not lie within the range bracketed by numbers x and y inclusive.
  • ∞: the property must be infinite (same as = _).
  • ?: the property must be unknown (same as = ?).
  • (In cases where special values like ∞ or Unknown (?) can occur, these can be directly selected from the selection list without having to type them in manually).

A few additional ones are also available for Cost in gliders:

  • Known: a synthesis is known to exist, but the exact number of gliders has not yet been computed.
  • Unknown: no synthesis is currently known (same as ≥ x).
  • Partial: a partial synthesis is known (same as > x).
  • TBD: the information about whether or not syntheses for this object is known or not has not yet been added to the database.

Any field where you need to enter a number gives you great flexibility in entering one. You may enter any of the following:

  • An empty field is treated as the default value (which is usually zero, except for some mathematical operators like multiplication and division).
  • Integers (e.g. 1), real numbers (e.g. 1.5), or scientific notation (e.g. 1.5e6).
  • Special fractions ¼ ½ and ¾ mixed numbers like 1½.
  • Infinity, by typing _ or ∞ or infinity.
  • Some mathematical constants (e, π, τ, ϕ).
  • For unknown values, such as gliders for objects without synthesis, frequency or rarity for uncommon objects, or heat for extremely large spaceships, enter ? or x or unknown. (Partial syntheses can be specified by forms like x+2.)
  • Some simple algebra is allowed (e.g. x+2, 1e6-1, 2π or 2*π or 2×π, 2/3, 3÷4, π^2 or π², √2, 3√10, 3!).
  • You can also enter multiple numbers separated by semicolons. With = the numbers are joined with or (e.g. = 1;2;4 means is 1 or 2 or 4). With any other comparison type, the numbers are joined with and (e.g. ≠ 1;2;4 means is not 1, and is not 2, and is not 4). Specifying multiple numbers for anything other than = or ≠ is not likely to be useful (e.g. < 1;2;3 is the same as < 1).

Gliders selects the minimum number of gliders needed to synthesize the pattern. You can also select Unknown for patterns with no known syntheses, Partial for patterns with simple syntheses based on an as-yet-unsynthesized predecessors, and TBD for patterns for which syntheses are planned, but have not yet been attempted, so their synthesis status is not yet known. You can also select Known for patterns for which syntheses have not yet been attempted, but it has been proven that syntheses definitely exist using standard techniques.

Gliders/bit selects the relative cost in gliders, i.e. number of gliders divided by minimum population. (This is treated as the same as the number of gliders in cases when the number of gliders is unknown, i.e. Unknown, Partial, TBD, or Known). Other pages on this site take particular interest of patterns for which this value =1 or >1.

Modulus selects a modulus that should be matched. This is usually the same as period, but may be period/2 for patterns with glide symmetry, and may also be period/4 for asymmetric patterns with 90° rotational glide symmetry. You can also qualify by the ratio of modulus to period.

Period selects the pattern’s repetition period. This is the minimum number of generations that it takes for the pattern to return to its original state. The meaning varies, depending on the pattern type:

  • Still-lifes, pseudo-still-lifes, and quasi-still-lifes always have a period of 1, so this criterion is ignored when only searching for them.
  • Oscillators, pseudo-oscillators, and quasi-oscillators return to their original state.
  • Spaceships, pseudo-spaceships, and quasi-spaceships return to their original state, translated in space.
  • Constellations contain multiple copies of the above, and are treated similarly.
  • Guns return to their original state, but also release spaceships.
  • Puffers and wick-stretchers return to their original state, translated in space, and leave behind periodic debris such as still-lifes, oscillators, or spaceships.
  • Breeders return to their original state, possibly translated in space, and leave behind growing debris such as guns, puffers, or wick-stretchers.
  • Methuselahs are unstable and aren’t periodic, so they have no periods. For search purposes, however, measurements of their period and other oscillating properties (such as modulus, heat, volatility, bounding box, etc.) are measurements of those properties for their final census.

Population (min.) selects patterns based on their minimum population or range of populations. If a pattern has multiple phases with different populations, the lowest population is used for search purposes, and is also the criterion used on the rest of this site for categorizing objects by population. For growing patterns, such as guns, puffers, wick-stretchers, and breeders, this is the population of their first recurring state, before they have produced any output.

Population (max.) selects patterns based on their maximum population or range of populations. If a pattern has multiple phases with different populations, this is the population of the largest phase. Growing patterns like guns, puffers, wick-stretchers, and breeders have infinite maximum populations. The maximum population of methuselahs is defined as the population of the largest phase of its final census.

Population (avg.) selects patterns based on their mean population or range of populations. If a pattern has multiple phases with different populations, this is the mean population of all phases. Growing patterns like guns, puffers, wick-stretchers, and breeders may have infinite average populations. The average population of a methuselah is defined as the mean of the populations of all phases of its final census.

Population (min./max.) selects patterns based on the ratio of minimum to maximum population. This will be 0 for infinitely growing patterns, and 1 for still-lifes, including the empty field.

Slope selects moving patterns based on their slope of movement. These vary from 0 for orthogonal movement to ½ for diagonal movement. It is not defined for non-moving patterns, and is ignored for them.

Velocity selects patterns based on their velocities or range of velocities. Velocities are described as fractions of c (the speed of light). Spaceships, pseudo-spaceships, quasi-spaceships, puffers, and wick-stretchers have non-zero velocities. Still-lifes, pseudo-still-lifes, quasi-still-lifes, oscillators, pseudo-oscillators, quasi-oscillators, and methuselahs have zero velocities. Constellations and breeders can be either. For non-zero velocities, you can also select a specific direction, or search all directions. (This is basically the same as selecting Slope, but is simpler.) Velocity is always measured along the major axis of movement; e.g. an oblique (2,1)c/6 spaceship would be indicated by 2c/6, or more simply, c/3.)

Uncommon

This section includes less common numeric constraints.

Active rotor cells is the number of cells in an oscillator that change. This is zero for still-lifes. It is infinite for moving and growing patterns. For pseudo-objects, quasi-objects, and constellations, this is the sum of the number of rotor cells in the component objects. For methuselahs, this is the number of active rotor cells in the final census.

Density (min/) is the ratio of minimum population and and influence. This is always between 0 and 1. In Life, this is always less than ½.

Density (max.) is the ratio of maximum population and and influence. This is always between 0 and 1.

Density (avg.) is the ratio of average population and and influence. This is always between 0 and 1. In Life, this is always less than ½ for still-lifes and period 2 oscillators.

E.F. (Evolutionary Factor) is the ratio of T.T.S. (Time to Stabilize) to minimum population. This is non-zero only for unstable patterns (e.g. methuselahs).

Frequency is the relative number of times an objects occurred in Achim Flammenkamp’s random soup experiments. It is 1/rarity. Except for a small number of common objects, most objects have frequencies of 0. This only applies to still-lifes and oscillators in the Life rule; all other categories are considered to have frequencies of N/A for search purposes.

Heat is the average number of unstable cells. It is zero for still-lifes, and can be infinite for growing patterns. For methuselahs, this is the heat of the final census.

Influence is the number of cells in a pattern’s area of influence. A cell is within a patterns area of influence it is is living cell at any time during any of the pattern’s phases, or is ever adjacent to any of a pattern’s living cells. This is infinite for moving or growing patterns. For methuselahs, this is the influence of the final census.

Number of rotors is the number of distinct disjoint rotors in an oscillator. This is 0 for still-lifes. This is 1 for basic oscillators, although in some cases e.g. Tumbler) two adjacent rotors interact without touching. For pseudo-objects, quasi-objects, and constellations, this is the sum of the number of rotors in the component objects. For methuselahs, this is the number of rotors in the final census.

Rarity is the inverse of Frequency. Except for a small number of common objects, most objects have infinite rarity. This only applies to still-lifes and oscillators in the Life rule; all other categories are considered to have rarities of N/A for search purposes.

Temperature is the ratio of heat to the number of active rotor cells. This is always between 0 and 1 inclusive. It is defined for moving and growing patterns, even though those have infinite temperature and active rotor cells.

Time to stabilize (T.T.S.) is the number of generations it takes the pattern to reach a periodic form. This is non-zero only for unstable patterns (e.g. methuselahs) and is measured from their smallest form.

Volatility is the number of active rotor cells divided by the minimum population. This always varies between 0 for still-lifes and 1 for phoenixes. This is also 1 for moving and expanding patterns.

Volatility (strict) is similar to Volatility, except only rotor cells whose period is the same as that of the whole pattern are considered. This always varies between 0 and the volatility.

Volatility (relative) is the ratio of Strict volatility to Volatility. This always varies between 0 (for patterns with pseudo-periods, e.g. with two independent rotors of different periods), and 1 (for patterns with non-trivial rotors, and all patterns whose periods are prime or 1).

Boxes

This section includes bounding boxes.

Box (hull) selects the bounding box that includes all generations. This is similar to the largest box, but may be slightly larger, and can be infinite for moving and growing patterns.

Box (largest) selects the largest bounding box the pattern fits in. For growing patterns (e.g. guns, puffers, wick-stretchers, and methuselahs that release gliders), this can be infinite. For methuselahs, this is the largest box achieved once the pattern stabilizes.

Box (smallest) selects the smallest bounding box the pattern fits in. For patterns with multiple generations (e.g. oscillators and spaceships), this may not necessarily be the size of the box in the pattern with the smallest population.

Rotor box is the size of the box that completely encloses all the rotor cells in a pattern. This is defined for oscillators. It is zero for still-lifes. For pseudo-objects, quasi-objects, and constellations, it is defined as the smallest box that encloses the rotors in all the component objects. For methuselahs, this is defined based on the final census.

Each of the above categories has six separate properties:

  • Width is the length of the shorter dimension.
  • Height is the length of the shorter dimension.
  • Area is the area of the box, i.e. width × height.
  • Circumference is the circumference of the box, i.e. 2 (width + height).
  • Diagonal is the length of the diagonal, i.e. √(width² + height²).
  • Squareness is the ratio of height/width. It varies from 0 for patterns that grow infinitely in one direction, to 1 for square patterns.

Text

This section includes constraints that are specified by text strings rather than numeric values.

Most of these include a drop-down list:

  • Any ignores this constraint.
  • Is matches the whole name.
  • Begins with matches the beginning of the name.
  • Ends with matches the end of the name.
  • Contains matches any part of the name.
  • Does not begin with, Does not end with, and Does not contain are similar, but they exclude all patterns whose names match as above, rather than include them.

Names can contain wildcards — a ? matches a single character, while a * matches any number of characters. A * at the beginning of a name is the same as Ends with, a * at the end of a name is the same as Begins with, and a * at both ends is the same as Contains. Multiple names can be separated by , or ; to indicate or. A ~ between names means and not, and at the front means not. For example, Is “~*jam*” is the same as Doesn’t contain “jam”, and Contains “snake~block~boat” means Contains “snake” but neither “block” nor “boat”.

APG name allows for searching for a pattern by its canonical apgcode, as used by apgsearch and Catagolue. This is well defined for still-lifes, oscillators, and spaceships. It is also well defined for pseudo-objects, quasi-objects, and constellations containing these, although apgsearch and Catagolue currently do not track them. This is not defined for constellations containing patterns of different velocities, or methuselahs. There is also a definition that covers patterns with linear growth (e.g. guns, puffers, and wick-stretchers), but this only describes the pattern’s growth signature, rather than the pattern’s image, so it is not necessarily unique. Because it is difficult to compute, it is provided for all puffers and guns in the pattern collection, but is not computed for patterns that are entered manually. APG names contain lower-case letters, digits, and underscores. All other punctuation is ignored.

File name allows for searching by a pattern’s file name. This is always a name from 1-8 characters long, containing digits, hyphens, and lower-case letters. It usually begins with a number, that is the minimum population, except in cases of growing patterns (e.g. guns, puffers, wick-stretchers, and breeders). Letter case is ignored. Other punctuation characters are ignored. If a path or extension is given, it is ignored (e.g. searching for /directory/5gl.rle will search for the name 5gl.

Glider number (all) allows for searching for specific gliders in David Eppstein’s glider repository, within the list of all gliders in all rules. Gliders are numbered sequentially, beginning with 1. Only certain iconic gliders are listed.

Glider number (rule) allows for searching for specific gliders in David Eppstein’s glider repository, within the currently-selected rule. Gliders are numbered sequentially in each rule, beginning with 1. Only certain iconic gliders are listed.

Header name allows for searching for patterns within specific groups within the pattern database. Header names contain digits, hyphens, and lower-case letters. Case is ignored. All other punctuation is ignored. These groups correspond to how data is arranged in various articles elsewhere on this site, and on how data is arranged in the internal database. This is normally only done for internal purposes, or for specialized pattern groups that aren’t easily qualified by category, period, and population. Sometimes, the name is intuitive (e.g. p3-lg is period 3 oscillators with populations ≥35, pp2-12 is 12-bit period 2 pseudo-oscillators, ss-2o5 is 2c/5 orthogonal spaceships, and 34-p8 is period 8 oscillators in B34/S34.)

Hickerson Rotor Descriptor (HRD) allows for searching oscillators and similar patterns by the signature of their rotors.

LifeWiki page allows for searching for patterns that refer to a particular page on the LifeWiki. Wiki names contain digits, lower- and upper-case letters, underscores (in place of spaces), and certain punctuation characters, including hyphens, apostrophes, $, and parentheses.

LIS name allows for searching for a pattern by its LIS encoding, which is the format used by Niemiec’s still-life search program, and is the format used by the database that runs this site and its search engine. This is only well-defined for patterns with sizes up to 223×223.

Neighborhoods allows for searching for all patterns that work within a certain group of related outer totalistic rules. If you enter a string of the form Baa-bb/Scc-dd, only patterns that require birth conditions aa but not bb and survival conditions cc but not dd, are matched. Conditions that are neither required nor forbidden may be either present or absent (e.g. B3/S23 matches B3/S23, B3/S238, B36/S23, B38/S23, and B38/S238).

Pattern name allows for searching for a pattern by its human-readable pattern name. Letters, digits, periods, $, and ü are used. Case is ignored. All other punctuation is ignored. The phrases with and w/ are considered equivalent.

SOF name allows for searching for a pattern by its SOF encoding, which is the canonical format used on pentadecathlon.com. This is well defined for still-lifes, oscillators, and spaceships. It is also well defined for pseudo-objects, quasi-objects, and constellations containing these, although pentadecathlon.com does not currently track them. This is not defined for constellations containing patterns of different velocities, methuselahs, or growing patterns. SOF names contain digits, lower- and upper-case letters, and any of the punctuation characters +-:;<=>?@[\]^_`{|}~ and ending with a period (which may be omitted, since it is always present in the format, so it is assumed).

Pattern image allows for searching for a pattern by its image. Because this image is usually large or two-dimensional, the entry field for the pattern is placed at the bottom of the search criteria. The pattern can be in one of several different input formats. You can specify an image format, or select Auto, in which case the format is automatically detected. Currently supported are APG, Cells, Life 1.05, Life 1.06, LIS, RLE, SOF. If the format is unknown, Cells is assumed. If this field is empty, the search is not constrained by pattern image. To explicitly search for an empty pattern, enter at least a comment or an empty cell. The window is initially tall enough to enter two lines of text, but you can adjust the window height to be larger, if desired.

The easiest way to enter a pattern is to copy and paste it from another Life program like Golly. Pattern orientation is not important. Phase is not important for oscillators, pseudo-oscillators, quasi-oscillators, spaceships, pseudo-spaceships, quasi-spaceships, or constellations. Extra spaces outside the pattern are ignored. The exact pattern must be specified (e.g. oscillators and spaceships must include their sparks). For methuselahs or growing patterns, the database only stores a small number of generations, so it may be more useful to search for them by category, as the lists of patterns in those categories are small.

A pattern can also be entered manually. Spaces, punctuation and line-drawing characters (- + and |) are treated as dead cells, and all others as living cells. Avoid using # characters at at the start of a line, or ! character anywhere, as these may cause the pattern to be interpreted as one of the special file formats.

When using Cells, RLE, or Life 1.05 formats (but not Life 1.06, APG, LIS, or SOF), the ? character has special meaning. Cells indicated by ? are wildcards — they match either living or dead cells. This allows one to search for specific pieces of patterns (e.g. patterns with an eater tail on one side, patterns with mold rotors, etc.) Note, however, that using wildcards to match pattern images is extremely time-consuming. It is generally faster to combine such wildcards with other constraints, especially when searching for oscillators or spaceships (e.g. when searching for patterns that contain mold rotors, one should also also specify period=4, or possibly period≥4, if one also wants to include period-multiplied oscillators).

Results

The search results are displayed as a list of patterns or a stamp page showing images of patterns below the search criteria table. Clicking on a pattern’s name or image selects it and displays detailed information about it. If the search finds exactly one result (which usually happens when one enters an image, or a pattern by name with no wildcards), that result is automatically selected.

The List view shows several columns of data. A column is shown for every search criterion that is selected, whether it contributes to the search or not. (E.g. to display the Heat of all patterns, select the Heat criterion, and set it to Any.). A column is also included for the criterion used for sorting. In any case, columns are always included for Minimum population, Period, Pattern name(s), and Image.

In the List view, clicking on a column heading changes the sort order, and clicking on a second time reverses the sort direction. If results from multiple rules are shown, clicking on a rule’s heading collapses the results for that rule, allowing easier access to results from other rules (especially if a large number of results have been found). Clicking on it a second time re-displays the hidden results.

Clicking on the pattern name or image selects the pattern, and displays details about that pattern in the table below. In addition, clicking on some other fields in the table have other effects:

  • Clicking on Category goes to the main category page for patterns in that category, or for non-Life rules, the section for such patterns.
  • For Life, clicking on the population goes to the page for all patterns with that population.
  • Oscillators and spaceships with glide symmetry show period and also the number of times the pattern recurs. For example, the Queen Bee shows a period of 30(/2), which means its period is 30, its modulus is 15, and it recurs twice during its cycle. Clicking on the period goes to the page for the main category page for patterns with that period, or for non-Life rules, the section for such patterns. This only works for still-lifes, pseudo-still-lifes, quasi-still-lifes, oscillators, pseudo-oscillators, quasi-oscillators, and constellations, and also methuselahs in Life.
  • Clicking on the velocity goes to the section for spaceships with that velocity
  • For Life, clicking on Gliders goes to the page for patterns that cost that number of gliders (2-7 or ≥100), or other groups that are tracked (i.e. 1 glider/bit, > 1 glider/bit, partial, unknown).
  • For Life, clicking on Gl./bit goes to the page for patterns that cost that number of gliders (i.e. 1 glider/bit, > 1 glider/bit, partial, unknown).
  • Clicking on Freq. or Rarity brings up the page for common objects.
  • For spaceships, clicking on Glider # or Gl. # (rule) brings up the Glider Repository page for all rules or this rule, respectively.
  • Clicking on APG or SOF name brings up the Catagolue or pentadecathlon page for this pattern, respectively.
  • Clicking on Header brings up the corresponding article on the page associated with the related category (or rule page, if not Life).
  • Clicking on File brings up the synthesis file.

In the Stamp view, clicking on a pattern selects it, and displays details about that pattern in the table below. Clicking on the top or bottom left corners of the stamp behave respectively as Page Up or Page Down to scroll through multiple results. Clicking on the bottom right corner opens an RLE file of the stamp page itself. Below the stamp is a standard stamp navigation bar with the following buttons:
  • ⇑ and ⇓ respectively go to the top and bottom of the page.
  • PNG opens the stamp page image.
  • RLE opens the RLE image of the stamp page.
  • APG, LIS, and SOF return lists of all search results in the selected format.

If there are too many results to fit on one page, navigation buttons are available above and below the search results to allow scrolling through the different pages.

Details

When a result pattern has been selected, its details are displayed below the search results. If a search only finds one pattern, that pattern is automatically selected. If a pattern search yields no results, the entered pattern may be shown with some information deduced about it, but most properties will not be shown.

The information is displayed in a two-column table, with property names on the left and property values on the right. Clicking on a property name usually loads a page for that property in general, while clicking on the property value usually loads a page for that property value in particular. Properties that are unknown or not applicable are omitted. The following properties may be displayed:

  • Rule is the rule that applies to this pattern. Clicking on the rule opens the page for that rule.
  • Name(s) is the name or names given to this pattern.
  • Smallest box is the box for the smallest phase of the pattern, or the smallest initial phase for methuselahs.
  • Largest box is the box for the largest phase of the pattern, or the largest census phase for methuselahs.
  • Hull is the box that includes all phases of the pattern, or all census phases for methuselahs. (Each of these boxes includes width × height, ▮ area, ▯ circumference, ⍂ diagonal, squareness (i.e. height/width));
  • Population shows the minimum, maximum, and average populations. For methuselahs, minimum population is for the smallest predecessor, and average and maximum populations are for the census. Clicking on this opens a page for all patterns of that population.
  • Symmetry shows the pattern’s symmetry and glide symmetry.
  • Category shows the pattern’s category. Clicking on this opens a page for all patterns of that category. If other sub-categories are appropriate, they are also listed here. (E.g. constellations may also be quasi-still-lifes; oscillators may also be flip-flops.)
  • Density shows the minimum, maximum, and average densities, i.e. ratios of corresponding populations and influence.
  • Frequency shows frequency of occurrence for common objects.
  • Gliders shows the minimum number of gliders required to synthesize the pattern. Clicking on this opens a page for all patterns with a similar cost in gliders.
  • Heat shows the pattern’s heat.
  • Influence shows the pattern’s influence.
  • Period shows the pattern’s period, and also modulus, if different. Clicking on this opens a page for all patterns with the same period.
  • Rule(s) shows the range of rules in which this pattern works. Note that this does not include rules where the pattern behaves similarly but has some phases with trivial differences.
  • Rotor shows information about an oscillator’s rotor, including the number of rotors, number of rotor cells, and the rotor box.
  • Temperature shows the pattern’s temperature.
  • T.T.S. (Time To Stabilize) shows how long it takes a methuselah to reach its final census, and also the E.F. (Evolutionary Factor), which is T.T.S./initial population.
  • Velocity shows the velocity of a moving pattern, and also its slope.
  • Volatility shows the volatility of a pattern, and also its strict volatility, if different.
  • Pattern: clicking on View pattern brings up an RLE file of the pattern itself.
  • Synthesis lists the synthesis file for the pattern. Clicking on this brings up the RLE file of the synthesis. Some patterns may have two synthesis files.
  • Multi-color lists the multi-color synthesis file for the pattern. Clicking on this brings up the RLE file of the multi-color synthesis.
  • APG shows the APG encoding of a pattern, as used by Catagolue. Clicking on this opens the Catagolue page for that pattern.
  • SOF shows the SOF encoding of a pattern, as used by pentadecathlon.com. Clicking on this opens the pentadecathlon.com page for that pattern.
  • LIS shows the LIS encoding of a pattern, as used internally by this site, and is used for ordering most of the pattern lists, including the sequences of still-lifes from 15 bits and up.
  • Gliders shows a spaceship’s glider number in David Eppstein’s Glider repository. Clicking on this opens up the repository page for the spaceship.
  • HRD Shows the Hickerson Rotor Descriptor for oscillators and similar patterns. Click See similar to repeat the current search but restricting the results to patterns with the same rotor descriptor.
  • LifeWiki shows one or more LifeWiki links related to this pattern. Clicking on them opens up the respective LifeWiki pages.

The pattern image is displayed below the details table. Huge patterns are not displayed, to avoid very long browser delays. (The only patterns currently affected are the huge spaceships and the Geminoid replicators.) The pattern is shown in a canonical representation (i.e. initial generation and specific orientation), which may differ from what was originally entered, if the search was by image.

There is a control bar below the pattern, with the following controls:

  • Export as lets you select a format for the Export function. This is one of RLE (the default), Cells (plain text), Life 1.05, Life 1.06, Apg, LIS, or SOF.
  • Export converts the pattern image into a text format, as selected above, and displays it in a box below the control bar. These can then be copied and pasted into a text editor or Life program.
  • Save Search saves all the search criteria on the page into a text string, and displays it in the box below the control bar. This lets you save complex search criteria by cutting and pasting them into a text file for later use.
  • Load Search loads search criteria previously saved by save Search from the text box.

Search for still-life

This section lets you search for still-lifes or difficult still-lifes in Life by number.

First, select the still-life size from the left drop-down list, from 4…24. Next, select the type of search from the middle list, either . (for still-life) or # (for difficult still-life). Finally, select the still-life number into the right text box and press Search. If the selected still-life has an associated synthesis file, that file is opened.

Still-lifes have numbers beginning with 1 (e.g. 14.1…619). All syntheses up to 14 bits are available.

Difficult still-lifes are defined from 14…18 bits. From 14…16 bits, they have numbers beginning with 1 (i.e. 14#1…4, 15#1…18, 16#1…94), and from 17…18 bits, they have numbers beginning with 100 (i.e. 17#100…398 and 18#100…790). All difficult syntheses up to 17 bits are available.

Search for rule

This section lets you search for outer totalistic and non-totalistic rules based on their arrangements of neighbors.

This section displays a cellular automata rule in four formats. Any change made in any one of the fields is immediately reflected in all the others.

At the top is a graphical map of all possible neighborhoods. The left half shows birth conditions, while the right half shows survival conditions. In each half, the numbers on the left indicate outer totalistic neighbors, while the images beside them show individual non-totalistic sub-neighborhoods. Clicking on a sub-neighborhood toggles inclusion of that sub-neighborhood, while clicking on a number toggles inclusion of all sub-neighborhoods that are subsets of one outer totalistic neighborhood. (For example, clicking on the grey 1 toggles birth on 1 neighbor in any position, while clicking on image beside it toggles birth on one diagonal neighbor.)

Below this is a text table of all possible neighborhoods. This is in the same format as the graphical map. Each neighborhood is shown in white if it is not selected, and green if it is selected. Outer totalistic neighborhoods for which some subsets are selected but others aren’t selected are shown in yellow. There are also three additional buttons at the top right. Undo reverses the action of the most recent button click. Moore and Hex switch between 8-cell Moore neighborhood and 6-cell hexagonal neighborhood.

In the middle is a RLE-file rule string, as supported by Golly or other Life programs. Most Life programs accept outer totalistic rule strings. Golly also accepts non-totalistic strings in the Moore neighborhood, and outer totalistic strings in the hexagonal neighborhood. (It does not yet accept non-totalistic strings in the hexagonal neighborhood; to use such rules with Golly, one must currently copy the rule table into a file, place it in Golly’s Rules directory, and use Golly’s Rule Loader to load it.) Golly also supports 4-cell Von Neumann neighborhoods, but this search tool does not, as it only supports rules with two states, and Von Neumann rules need more than two states to be interesting.

The rule string is typically of the form Bbbb/Ssss, where bbb is a list of 0 or more digits representing birth conditions, and sss is a list of 0 or more digits representing survival conditions. For non-totalistic rules, each digit may be followed by one or more letters that describe specific sub-neighborhoods to be included. If most are included, it is easier to specify the digits, a minus sign, and a list of sub-neighborhoods to be excluded instead. Finally, an H at the end indicates that a hexagonal neighborhood is used, rather than the default Moore neighborhood. More details about this format can be found in Golly’s documentation.

On input, this field has a fair bit of flexibility. Letter case is ignored. The / may be omitted. The digits and sub-neighborhood letters may be specified in any order, and need not be unique. The legacy format of sss/bbb is also supported. Once any change is made in either of the two rule tables above, this rule string is re-generated in its canonical form.

Directly below the RLE rule is a MAP rule that describes the 512 possible Moore neighborhoods or 128 possible hexagonal neighborhoods, encoded in base64 encoding. (This field is read-only, and cannot be used to convert MAP rules into RLE rule strings.)

At the bottom is a rule table, suitable for use by Golly’s Rule Loader. To use this table, copy it to a text file, and save it in Golly’s Rules directory. (This field is read-only, and cannot be used to convert Golly rule tables into RLE rule strings.) The description of the format used for this table can be found in Golly’s documentation.

Below the rule string is a link to the Catagolue search results for the selected rule. (This page will be empty, if no searches for that rule have yet been performed).

If a rule is outer totalistic, an additional link is provided to David Eppstein’s glider repository, showing all a list of spaceships listed there for the selected rule. Note that that list is merely representative, rather than exhaustive. In rules that have been studied extensively (such as Life), there may be many additional known spaceships that are not listed there.

Search Catagolue

This section lets you search Catagolue.

In the first line, select the type of search to perform:

  • Attribute: This is similar to Object, but displays any additional attribute information associated with a particular object.
  • Census: This is the most commonly selected search. It provides census information about searches for a rule, a symmetry within a rule, or a category of objects within a symmetry result. From a category, you can get a list of objects found in that category.
  • Haul: This provides a list of hauls related to a particular rule and symmetry. You must specify whether you want to see hauls that have not yet been committed, hauls that has not yet been verified, or hauls that have been verified. You can also limit the list to hauls submitted by a particular user.
  • Home: This shows the Catagolue home page.
  • Object: By entering an apgcode for an object, you can get a list of all soups found that contain that object. You can also search via the apgsearch of a pattern found by a previous Pattern Search.
  • Object Count: This is like Census, but just shows the number of objects found in each category in an editable text format arranged as a CSV table.
  • Object List: This is like Census, but just shows a list of all objects found in an editable text format as a CSV table.
  • Rules: This points to the Catagolue rules page.
  • Statistics: This shows statistics about searches in the Life rule.
  • Synthesis List and Synthesis Table: These display census results for a specific pattern type within a given rule. These are respectively similar to Object List and Census, except the patterns are sorted in order of decreasing synthesis cost, in three groups: first, patterns without syntheses (showing a cost of infinity), objects, and non-objects (including pseudo-objects, quasi-objects, and constellations).
  • User: This shows information about contributions by a specific user.

The other lines specify other search criteria, and are only shown if they apply to the search being performed.

If you are performing a User or Haul search, you should enter the user’s nickname or email address in the Name field.

If you are performing an Object or Attribute search, you should select Here from the drop box and enter the APG code of the object. If you had previously used the pattern search to find the object, select Pattern search from the drop box and the result of that search will be used instead.

If you are performing a Census, Object, Object List, Object Count, Synthesis List, or Synthesis Table search, you must select a rule from the drop down list. If the rule you want is not on the list, select Other and type in a rule. You can select From Search rule… to use the rule determined by the Search Rule section above. You can select Any in a Census search, in which case a list of the most recently searched rules is shown.

If you are performing a Census, Object, Object List or Object Count search, you must select a symmetry from the drop down list. If you want to select a non-standard symmetry, select Other and type in the symmetry. You can select Any in a Census search, in which case a list of all common symmetries is shown.

If you are performing a Census search, you can select a date, by selecting year, month, and day from drop down lists. If you specify a date, statistics are displayed as of that date. If you select Any, the most recent statistics are displayed.

If you are performing a Census search, you should select an object type from the drop down list. If you select Any, a list of all categories for which objects have been found is shown. If you select a category, you must provide additional information:

  • For Still-lifes, you must specify the population.
  • For oscillators and spaceships, you must specify the period.
  • For puffers and guns, you must specify the repeat factor. (This is usually the modulus, but is always a fraction of the period.)
  • The other categories (e.g. over-sized patterns, growing patterns, and pathological patterns) lump all objects into a single list, so they do not require any additional information.

At the bottom of the table is a link to the desired Catagolue search results, if sufficient information has been supplied, or an error message indicating what information is missing.

Tips

Some helpful search tips:

  • Normally, the list of patterns searched include all patterns on this site that occur in detailed object lists, currently 14880 items. You can also optionally search through around 107058 additional items that appear on expanded object stamp pages (i.e. those that include 2000 or more patterns, without syntheses, 1000 per page). To avoid long delays during page loads, this option is on a separate page, Expanded search. On that version of the search page, the inclusion of these additional patterns can be enabled by checking the check-box Include expanded object lists. If this is disabled, the search will be as limited as (and as fast as) the regular search.
    • Normal search includes all Life still-lifes up to 15 bits, pseudo-still-lifes up to 16 bits, period 2 oscillators up to 18 bits, and period 2 pseudo-oscillators up to 18 bits.
    • Expanded search includes all Life still-lifes and pseudo-still-lifes up to 18 bits (and symmetric ones up to 24 bits), and period 2 oscillators and pseudo-oscillators up to 21 bits.
    • All other object types, and all other rules, are the same in both kinds of search.
  • If you get too many results, try to narrow down the search. When searching by name, use multiple wildcards. (For example, when looking for snakes on candlefrobras, try *snake*frob* rather than just *frob*. It can also be helpful to narrow the search by specifying a period, a range for minimum population, and/or a velocity for spaceships.
  • If you get too few results, or none at all, try removing one search criterion at a time until some results are found.
  • Four properties (minimum population, period, name, and image) are always displayed in the search results table. Other properties are only displayed if you supply search criteria for them, or if you specifically sort based on those properties. To include other properties in the table without actually constraining them, enable a constraint and set it to Any.
  • Search speed is increased if you specify specific rules, categories, populations, periods, or velocities, as these criteria automatically reject entire groups of patterns without needing to examine each one separately. Note that searching by pattern image also filters Life still-lifes, pseudo-still-lifes, and quasi-still-lifes by implicitly selecting Minimum population ≥ (population of the entered image, assuming all wildcards are dead) and Maximum population ≤ (population of the entered image, assuming all wildcards are alive). The fastest search is to search by name for one specific still-life, e.g. 12.121; such searches should be virtually instantaneous.
  • Text representations (i.e. Apg, LIS, and SOF names) are not stored in the database, but each pattern’s representation is calculated the first time it is needed. Thus, searches by name (e.g. Apg begins with xp3_ can initially take a long time, unless other constraints are also supplied. Apg search names also contain implicit constraints for category, period, and population, unless complicated names are used (e.g. anything other than is, e.g. wildcards, multiple names, exclusions). Searching apg search names by image, however, is faster than a normal image search, because the apg search name also implicitly filtered by category and period.
  • Image search with wildcards is extremely time-consuming, so it is usually a good idea to provide as many other known criteria as possible (e.g. category, period).
  • It is easy to convert from one Life file format to another. Simply use the image search, paste the input file (which can be in any supported format) into the image text box, press Search, then use the Export function to export it into any other supported format.

Sometimes, results can be correct, although they may seem confusing or counter-intuitive:

  • When entering numeric expressions, the * for multiplication can sometimes be omitted, but not always; omitting it can sometimes yield unexpected results. It can always be omitted before π, τ and ϕ (e.g. 2π is 2*π). It can be omitted before e (e.g. 3e^2 is 3*e²), but not if it’s followed by + or –, as that indicates scientific notation (e.g. 4e-2 = 0.04, not 4*e-2). It can never be omitted with √ (e.g. 4√2 = fourth root of 2, not 4*√2). It may also never be omitted in any other cases not mentioned here.
  • In numeric expressions, implicit multiplication has a higher binding precedence than exponentiation. Thus, 4π² and 4π^2 are (4*π)² rather than 4*(π²). This may be counter-intuitive (and is, in this example, incorrect) but it is not likely to be fixed soon.
  • The displayed form of an object is the base canonical form that included in the object database. This may differ from what is entered via image search. In most cases, width ≥ height, but for moving patterns (e.g. spaceships, pseudo-spaceships, quasi-spaceships, and puffers), movement is always in the up and/or left directions, and velocity up ≥ velocity left.
  • Adding additional constraints can sometimes produce more results rather than less. This often happens when one constraint renders another meaningless, causing it to be ignored. For example, when searching for objects with velocity > 0, there are hundreds of objects (spaceships and puffers), yet when the search is also restricted to only still-lifes, there are thousands of results. This is because velocity is meaningless for still-lifes, so it is ignored, and all still-lifes are found.
  • Many properties (such as heat and volatility) are formally defined only for periodic patterns, such as oscillators (and, implicitly, still-lifes that have heat and volatility zero). Here, these definitions are extended to other kinds of objects, by being treated asymptotically, as limits as time goes to infinity (e.g. limx→∞ (statistics from generations xp…2xp-1, where p is the pattern’s period, or 1 if the period is not regular). This allows the definitions to be seamlessly extended to many other classes of objects:
    • Spaceships are completely volatile (e.g. heat is finite, and volatility and strict volatility are 1).
    • Heat and volatility of guns are dominated by their outputs (e.g. heat is infinite; volatility is that of their outputs, e.g. 1).
    • Heat and volatility of puffers and wick-stretchers are dominated by their outputs (e.g. heat is that of the puffer if output is all still-lifes, and infinite otherwise; volatility is that of the output).
    • Similarly, heat and volatility of breeders are dominated by their guns or puffers, and they in turn by their outputs. (e.g. heat is always infinite (e.g. ∞ for MMS breeders, and ∞² for all others).
    • Temperature for infinitely-expanding patterns can be meaningful even if heat is not (e.g. for guns, puffers, and breeders, it will be that of their dominating outputs.)
    • Methuselahs are dominated by their final census.
    • Irregular infinitely-expanding patterns (typically in rules other than Life) have infinite heat and volatility, zero strict volatility, and indeterminate temperature.

Known issues

  • Heat and volatility may be incorrect for methuselahs and growing patterns, and is meaningless for not quite stable objects.
  • Most periodic patterns can be found by image search in any phase. However, some large patterns are only listed by their main phases. These include methuselahs and expanding patterns (guns, puffers, wick-stretchers and breeders).
  • Huge patterns frequently lack accurate information about certain statistics, such as pattern image, average and maximum population, heat, large bounding box, hull, etc. (The only patterns currently affected are the huge spaceships and the Geminoid replicators.)
  • Wildcard image searches may take an extremely long time, especially if the image being searched for is fairly large, and there are no other constraints. It is best to apply as many other constraints as possible to first narrow the search.
  • Search by image does not actually analyze a pattern’s behavior (other than to check if it is stable), so if a pattern is not in the list of known patterns, most of its properties will not be available. In particular, the associated Catagolue link will only be correct if the pattern is a still-life. If it is anything else, its Catagolue link assumes a prefix of xp0_, for generic unstable pattern, unless a specific search category has been selected, and an additional constraint (i.e. population for still-lifes, pseudo-still-lifes, quasi-lifes, or constellations containing the same; period for oscillators, pseudo-oscillators, quasi-oscillators, spaceships, pseudo-spaceships, quasi-spaceships, or constellations containing these).

Browser issues

  • Some browsers display text more widely than others, which can sometimes cause entries in tables to wrap around.
  • Opera version 9 does not resize the pattern image properly. Also, in the list view, when a large number of results are found (for example, around 3000 or more), the sort order can be unpredictable.
  • Older browsers (such as this one) that lack support for HTML canvases do not support the stamp view of search results, thumbnail images in the list view of search results, or pattern images below search details.)
  • On some browsers, re-loading the page resets all form data. On some others, form data is preserved. In any case, search results are not preserved.

Privacy policy

All searching is performed entirely in the browser; no information is sent to any server, so no data is collected for any purpose whatsoever. (Not that it likely that anyone would consider Life patterns to be highly sensitive personal information anyway.) Some search results provide links that go to other sites like Catagolue or David Eppstein’s glider repository. Following those links to other sites may be subject to those sites’s privacy policies.

Lambda 演算与图灵完备性

发表于 2022-07-10

C++11 与 C++98/03 相比，带来了许多改进。这些改进中，离我们最近的莫过于从函数式编程语言中学来的 lambda 表达式。我想写过几行 Python 的人都知道 lambda 表达式的厉害之处。然而，尽管函数式编程是过去几年才流行起来的，但它的渊源可以追溯到电子计算机之前！

要理解 lambda 表达式的本质，让我们来玩一个叫做 lambda 演算的游戏。

假设我们有两种箱子，我们暂且称之为“潜影盒”和“末影箱”（与 Minecraft 中的潜影盒和末影箱不完全相同，但不影响理解），潜影箱可以嵌套其他的箱子，而末影箱不能。这些箱子有的有不同的颜色，有的是无色的。下面我们用字母表示颜色（无色用 W 表示。无色箱子仅起到一个“捆绑多个箱子”的作用，当它仅包含一个箱子时，就可以视为它包含的箱子本身），大写表示该箱子是一个潜影盒，小写表示该箱子是一个末影箱。箭头表示的是箱子的树形结构，跟 Minecraft 中一样，箱子中的物品是有序的。游戏的目标是通过下述操作，让箱子的嵌套关系变得尽量简单。

这个很麻烦的游戏有什么意义，它和一开始我们所提到的函数式编程又有什么关系？事实上，我们将会看到，正如 Minecraft 中可以用红石电路造出一台计算机，Super Mario 中也可以通过设计关卡来自动进行四则运算一样，只要合理设计箱子的内容，这个游戏可以做很多有趣的事情。至于它和函数式编程的关系，我们稍后再讲。
